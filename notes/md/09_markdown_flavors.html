Markdown Flavors: CommonMark vs GitHub Flavored Markdown (GFM)

Imagine you are writing a recipe. You could write it in a very strict, formal way, or you could write it casually, adding notes and special instructions just for yourself. Markdown is like the basic set of rules for writing text quickly on the web—it lets you use simple symbols to format things like headings and lists.

But sometimes, when you want to share that recipe with a specific audience (like a coding team or a social media platform), you need extra features. That’s where flavors come in. CommonMark is the basic, universal rulebook, while GitHub Flavored Markdown (GFM) is like adding specialized tools and shortcuts specifically designed for projects hosted on GitHub.

What is Markdown? The Foundation

Markdown itself is just a lightweight way to format text using simple, easy-to-read syntax. It’s not a programming language; it’s a plain text format that you can read easily and convert into nicely formatted HTML.

Think of Markdown as the basic alphabet—the letters A, B, C. You can form simple words with them. CommonMark is like the official dictionary defining exactly how those letters should be used to form standard English words. GFM is like taking that dictionary and adding specialized vocabulary for a specific context, like adding terms for cooking or coding into the same language.

CommonMark: The Universal Standard

CommonMark is an effort to create a single, unambiguous specification for writing Markdown. Its goal is to make sure that no matter which Markdown processor you use—whether it’s on a blog, a documentation site, or a simple text editor—the resulting formatted output looks exactly the same.

GitHub Flavored Markdown (GFM): The Feature Extension

GFM is not a replacement for CommonMark; it’s an extension. It takes the core rules of CommonMark and adds extra features that are particularly useful in environments like GitHub, where people often need to write technical notes, issue trackers, or README files.

Key Differences at a Glance

Feature CommonMark GitHub Flavored Markdown (GFM)
Purpose To define a universal, unambiguous standard for Markdown syntax. To extend Markdown with features specifically useful for GitHub and related platforms.
Scope Core Markdown elements (headings, lists, links, emphasis). All CommonMark features PLUS extra syntax for tables, task lists, and code blocks.
Code Blocks Basic fenced code blocks (using triple backticks). Adds support for syntax highlighting within code blocks, making code much easier to read.
Task Lists No native support for task lists. Introduces the `- [ ]` syntax for creating interactive checklists directly in Markdown.
If CommonMark is the basic grammar rules for writing any sentence, GFM is like taking those rules and adding specialized vocabulary—like specific terms for tracking bugs or organizing project steps—so you can write highly functional documentation specifically for a team environment.
